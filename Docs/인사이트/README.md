## 🍔 핵심 개념: 맥도날드 시스템

```
맥도날드 알바 = AI 에이전트 (Cursor, Claude)
├─ 전문 셰프 아니어도 OK
├─ 200줄 매뉴얼만 읽고 즉시 작업
└─ 적은 컨텍스트로 정확하게 일함

지점장 = 바이브 코더 (개발자)
├─ AI 에이전트 관리
├─ "왜"를 기록
└─ 의사결정 및 우선순위

본사 = 테크리드
├─ 매뉴얼 제공
├─ 표준 워크플로우 설계
└─ 전체 시스템 관리
```

---

## ⭐ 3가지 핵심 원칙

### 1️⃣ 200줄 원칙
- AI 에이전트가 200줄 읽고 즉시 작업 시작
- 200줄 넘으면 가이드 분할 필요
- 상세 내용은 Appendix로 분리

### 2️⃣ 컨텍스트 휘발
- 한 작업 끝나면 AI는 컨텍스트 잊음
- 리포지토리에 없는 정보 = 휘발됨
- **중요한 결정은 무조건 문서화**

### 3️⃣ "왜(Why)" 강제 기록
- 코드만 남고 "왜"는 사라짐 → 레거시
- PRD + ADR에 결정 이유 필수 기록
- 3개월 후에도 의도 파악 가능
