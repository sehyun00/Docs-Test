## 📋 화면 개요

| 항목               | 내용                                    |
| ------------------ | --------------------------------------- |
| **화면명**         | 종목 상세                              |
| **목적**           | 개별 종목의 상세 정보 및 차트 확인     |
| **진입 경로**      | 포트폴리오 상세 → 종목 카드 탭        |
| **이동 가능 화면** | ← 포트폴리오 상세, → 종목 편집 모달   |

### 사용자 스토리

> "투자자로서 나는 개별 종목의 **현재가, 보유 현황, 차트**를 한 화면에서 확인하고 싶다."

## 💡 논의 사항: Modal vs Page

### Option A: Modal (현재 프로토타입)

```
장점:
- 빠른 진입/탈출
- 맥락 유지 (뒤에 상세 화면 보임)

단점:
- 공간 제약 (차트 표시 어려움)
- 데이터가 많을 경우 스크롤 불편
```

### Option B: Page (별도 화면) ✅ 권장

```
장점:
- 충분한 공간 (주가 차트, 상세 정보)
- 모바일 UX 관례 (증권사 앱 참고)
- 확장성 (뉴스, 공시 등 추가 가능)

단점:
- 화면 전환 필요
- 뒤로가기 필요
```

**권장**: **Page 방식**으로 구현하여 미래 확장성 확보
