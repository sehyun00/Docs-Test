## 📋 화면 개요

| 항목               | 내용                                    |
| ------------------ | --------------------------------------- |
| **화면명**         | 앱 소개 온보딩                         |
| **목적**           | 신규 사용자에게 앱의 핵심 가치 전달    |
| **진입 경로**      | 앱 최초 실행 (비회원 상태)             |
| **이동 가능 화면** | → 로그인/회원가입                      |

## 💡 설계 가이드라인

> **PRD 참조**: `features/ui-design.md` - 온보딩 슬라이드 정의

### 1. 슬라이드 구성 (PRD 기준)

| 슬라이드 | 내용 | 목적 |
|---------|------|------|
| **1. 앱 핵심 가치** | 리밸런싱의 중요성 설명 | 앱의 존재 이유 전달 |
| **2. 포트폴리오 생성** | 포트폴리오 생성 방법 요약 | 첫 번째 행동 유도 |
| **3. 종목 추가/비중 설정** | 종목 추가 및 목표 비중 설정 방법 | 핵심 기능 소개 |
| **4. 시작하기** | CTA 버튼으로 앱 시작 | 최종 전환 유도 |

### 2. 형태

- **Carousel 형태**: 스와이프 가능한 카드 뉴스 스타일
- **진입 시점**: 앱 최초 실행 시 1회 노출

### 3. 디자인 고려사항

| 항목 | 권장 |
| ---- | ---- |
| 슬라이드 수 | 3~4개 (PRD 기준 4개) |
| 애니메이션 | 페이드 또는 슬라이드 전환 |
| 일러스트 | 앱 테마 색상과 일관성 유지 |
| 하단 인디케이터 | 도트 형태 (현재 위치 표시) |

### 3. 건너뛰기 동작

- 모든 슬라이드에서 "건너뛰기" 버튼 노출
- 건너뛰기 시 즉시 로그인 화면으로 이동
- 온보딩 완료 여부 로컬 저장 (다시 보지 않기)
