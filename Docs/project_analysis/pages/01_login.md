# 화면 기획서: 로그인/회원가입

> **Screen ID**: `screen-login`  
> **Phase**: 1 (MVP)  
> **우선순위**: P1  
> **마지막 수정**: 2026-01-04

---

## 📋 화면 개요

| 항목 | 내용 |
|-----|------|
| **화면명** | 로그인/회원가입 |
| **목적** | 앱 진입 및 사용자 인증 |
| **진입 경로** | 앱 최초 실행, 로그아웃 후 |
| **이동 가능 화면** | → 프로필 설정 (신규), → 홈 (기존 사용자) |

### 사용자 스토리
>
> "투자자로서 나는 **간편하게 앱에 로그인**하여 포트폴리오 관리를 시작하고 싶다."

| 니즈 | 중요도 | 현재 충족 여부 |
|-----|--------|--------------|
| 빠르고 간편한 시작 | ⭐⭐⭐ | ✅ (Google 로그인) |

| 신뢰감 형성 | ⭐⭐ | ⚠️ (브랜딩 강화 필요) |

## 💡 개선 아이디어

### 1. 브랜딩 강화

| 현재 | 제안 | 기대 효과 |
|-----|------|----------|
| 📊 이모지 로고 | 커스텀 앱 아이콘 | 전문성, 신뢰감 ↑ |
| 텍스트만 있음 | 미묘한 배경 애니메이션 (주가 그래프 실루엣) | 시각적 매력 ↑ |

### 2. 온보딩 슬라이드 (선택적)

```
[슬라이드 1] 포트폴리오를 한눈에
[슬라이드 2] 실시간 시세 확인
[슬라이드 3] 리밸런싱 자동 제안
         → [시작하기]
```

- **장점**: 기능 이해도 향상
- **단점**: 진입 시간 증가
- **결론**: MVP에서는 생략, Phase 2에서 A/B 테스트

### 3. 로그인 플로우 개선

```
현재:  [Google 시작] → 프로필 설정 → 홈
                    ↳ (기존) → 홈

제안:  [Google 시작] → 로딩 (0.5s) → 분기 자동 판단
                                   ↳ 신규 → 프로필
                                   ↳ 기존 → 홈
```

## ⚠️ 엣지 케이스 및 에러 상태

| 상황 | 현재 처리 | 권장 처리 |
|-----|----------|----------|
| 네트워크 오류 | ❌ 없음 | 토스트 "연결 실패. 다시 시도해주세요" |
| Google 로그인 취소 | ❌ 없음 | 토스트 "로그인이 취소되었습니다" |
| 서버 점검 중 | ❌ 없음 | 팝업 "서버 점검 중입니다 (예상 시간: HH:MM)" |
| 로그인 세션 만료 | ❌ 없음 | 자동 이 화면으로 이동 + 토스트 |

## 🔄 화면 상태 (States)

### 1. Default State

- 모든 요소 정상 표시
- Google 버튼 활성화

### 2. Loading State

```
┌───────────────────────────────┐
│ 🔷 로그인 중...  ⟳           │
└───────────────────────────────┘
```

- 버튼에 스피너 표시
- 버튼 비활성화 (중복 클릭 방지)

### 3. Error State

- 버튼 정상 복구
- 토스트 메시지 3초간 표시
