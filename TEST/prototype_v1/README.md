# Stock-Keeper UI Prototype V1

주식 리밸런싱 앱의 **최초 UI 프로토타입**입니다.  
단일 HTML 파일에 모든 화면과 컴포넌트가 포함된 모놀리식 구조로 빠른 프로토타이핑에 초점을 맞췄습니다.

---

## 🚀 실행 방법

- VS Code **Live Server** 확장으로 `index.html` 실행 권장
- 또는 브라우저에서 `index.html` 직접 열기

---

## 📱 화면 구성

### 1. 로그인 (`#screen-login`)

앱의 첫 진입 화면입니다. Google 소셜 로그인 버튼과 함께 앱의 주요 기능(포트폴리오 관리, 실시간 시세, 리밸런싱 분석)을 하이라이트로 노출합니다. 기존 사용자를 위한 "로그인하기" 링크도 제공합니다.

### 2. 프로필 설정 (`#screen-profile`)

신규 사용자의 프로필을 입력받는 온보딩 화면입니다. 프로필 사진 플레이스홀더, 닉네임 입력, 투자 성향(안정형/중립형/공격형) 선택 칩을 제공합니다. 칩은 단일 선택 방식으로 동작합니다.

### 3. 홈 - 포트폴리오 목록 (`#screen-home`)

메인 대시보드 역할을 합니다. 상단에 사용자 인사말과 총 평가금액 요약 카드를 표시하고, 하단에 포트폴리오 카드 리스트를 나열합니다. 각 카드에는 포트폴리오명, 종목 수, 평가금액, 수익률이 표시되며 "리밸런싱 필요" 뱃지가 조건부로 노출됩니다. FAB(+) 버튼으로 새 포트폴리오를 추가할 수 있습니다.

### 4. 포트폴리오 상세 (`#screen-detail`)

선택한 포트폴리오의 상세 정보를 보여줍니다. 총 평가금액/매입금액/수익금/수익률 요약 그리드와 함께 보유 종목 테이블을 표시합니다. 각 종목 행에는 현재 비율 vs 목표 비율을 시각화하는 미니 바 차트가 포함되어 있습니다. 하단에 "종목 추가"와 "리밸런싱 분석" 버튼이 있습니다.

### 5. 종목 검색 (`#screen-search`)

포트폴리오에 새 종목을 추가하기 위한 검색 화면입니다. 검색창에 종목명/코드를 입력하면 결과 리스트가 표시됩니다. 각 결과 아이템에는 종목명, 코드, 현재가, 등락률이 포함되며 클릭 시 종목 추가 모달이 열립니다.

### 6. 리밸런싱 분석 (`#screen-rebalance`)

포트폴리오의 현재 비율과 목표 비율을 비교 분석하는 화면입니다. 가로 막대 차트로 각 종목의 현재/목표 비율을 시각화하고, 하단에 매도/매수 제안 카드를 리스트로 보여줍니다. 각 제안 카드에는 현재 비율, 목표 비율, 권장 수량, 예상 금액이 표시됩니다.

### 7. 설정 (`#screen-settings`)

앱 설정 화면입니다. 프로필 정보 표시, 알림 설정(토글/주기/시간), 앱 정보(버전/이용약관/개인정보처리방침/오픈소스 라이선스) 그룹으로 구성됩니다. 최하단에 로그아웃 버튼이 있습니다.

## 🧩 주요 컴포넌트

### 모달

| 모달 ID | 용도 |
|---------|------|
| `stock-add-modal` | 검색된 종목을 포트폴리오에 추가. 보유 수량 입력(+/- 스테퍼)과 목표 비율 슬라이더 제공 |
| `threshold-modal` | 리밸런싱 임계값 설정. ±3%/±5%/±10% 라디오 옵션 |
| `add-portfolio-modal` | 새 포트폴리오 생성. 이름, 설명(선택), 색상 팔레트 선택 |

### 공통 UI 요소

- **화면 네비게이터**: 우측 하단 고정 패널. 숫자 버튼(1~7)으로 각 화면 직접 이동
- **헤더**: 뒤로가기 버튼 + 타이틀 + 우측 액션 버튼 구조
- **카드**: 그라데이션 컬러바 + 콘텐츠 레이아웃
